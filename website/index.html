Why LabWC Android?

Unlike X11-based setups that suffer from latency and overhead, LabWC Android provides a native Wayland pipeline directly to your Android display hardware.

⚡

Zero-Copy GPU Rendering

AHardwareBuffer integration with Android's SurfaceControl delivers frames directly to your screen without copy overhead.

🎨

Native Wayland Compositor

labwc provides a full Wayland compositor that runs natively on Android, supporting modern desktop features.

🐧

Arch Linux ARM

Full Arch Linux ARM environment with thousands of packages available via pacman. LXQt desktop included.

🔒

Proot Sandboxed

Containerized environment keeps your Android system clean while providing full Linux capabilities.

🎵

Audio Support

PulseAudio integration provides seamless audio output through the Android audio stack.

🚀

One-Click Setup

Single command installation handles everything from package installation to desktop configuration.

Architecture Overview

LabWC Android uses a multi-layer approach to deliver desktop Linux on Android:

1

Termux (Host Layer)

The foundation running directly on Android. Handles audio (PulseAudio), display (termux-x11), and GPU (virgl) services.

2

Proot Container

Arch Linux ARM container running via proot-distro. Sandboxed environment with full Linux capabilities.

3

Wayland Compositor

labwc runs inside the container as the window manager, communicating with wlroots-android-bridge for Android display access.

4

Desktop Environment

LXQt components (tint2, rofi, dunst, pcmanfm) provide a complete desktop experience.

Quick Start Guide

Get up and running in under 10 minutes. This guide assumes you have Termux installed from F-Droid or the official site.

Prerequisites

  • Android device with ARM64 processor (recommended 4GB+ RAM)
  • Termux app installed (v0.118.0+ from F-Droid recommended)
  • Termux-X11 app installed (for display output)
  • At least 3GB free storage space
  • Internet connection for package downloads

Step 1: Clone the Repository

# Clone the project
git clone https://github.com/yourusername/labwc-android.git
cd labwc-android

Step 2: Run the Setup Script

# Run the complete setup
bash setup.sh

This will automatically:

  • Install all required Termux packages
  • Create host-side shortcut commands
  • Install Arch Linux ARM proot container
  • Configure the desktop environment inside the container

Step 3: Start the Desktop

# Start all host services (audio, display, GPU)
start-all

# Enter the Arch Linux container
proot-distro login archlinuxarm --shared-tmp

# Start the desktop (inside the container)
desktop start

Step 4: Open Termux-X11

Open the Termux-X11 app on your Android device. You should see the full desktop environment running.

Commands Reference

Host-Side Commands (Termux)

CommandDescription
start-allStart all host services (audio, display, GPU)
start-audioStart PulseAudio server
start-displayStart X11 display and picom compositor
start-graphicsStart VirGL GPU acceleration
start-waylandStart Wayland compositor bridge
fix-desktopFull environment recovery and process cleanup

Container Commands (Arch Linux)

CommandDescription
desktop startStart the desktop environment
desktop stopStop the desktop environment
desktop restartRestart the desktop environment
desktop statusCheck desktop status

System Commands

# Update Arch Linux container
pacman -Syu

# Install new packages
pacman -S <package-name>

# Install GUI applications
pacman -S firefox gimp libreoffice

# Check running services
ps aux | grep -E "labwc|tint2|dunst"

# View desktop logs
journalctl -u desktop

Troubleshooting

Desktop doesn't start / black screen

Solution: Run the fix command and restart:

# On Termux host
fix-desktop
# Then re-enter container and start
proot-distro login archlinuxarm --shared-tmp
desktop start
No audio output

Solution: Restart PulseAudio and verify it's running:

# On Termux host
pkill -9 -f pulseaudio
pulseaudio --start --exit-idle-time=-1 --system=false \
    --load="module-native-protocol-tcp auth-ip-acl=127.0.0.1 auth-anonymous=1" \
    --disable-shm=yes

# Inside container, set environment
export PULSE_SERVER=127.0.0.1
GPU acceleration not working

Solution: Verify VirGL is running and check environment variables:

# On Termux host
ps aux | grep virgl

# Inside container, verify Mesa config
echo $GALLIUM_DRIVER  # Should show: virpipe
echo $MESA_GL_VERSION_OVERRIDE  # Should show: 4.0
glxinfo | grep "OpenGL renderer"
Package installation fails (pacman errors)

Solution: Fix pacman sandbox settings and reinitialize:

# Inside container
pacman-key --init
pacman-key --populate archlinuxarm
pacman -Syu
Proot installation fails

Solution: Clear proot state and reinstall:

# On Termux host
pkill -9 -f proot
proot-distro remove archlinuxarm
proot-distro install archlinuxarm

Frequently Asked Questions

What Android versions are supported?

LabWC Android works on Android 7.0 (Nougat) and above. For best performance, we recommend Android 10+ with at least 4GB RAM.

Do I need root access?

No. LabWC Android uses proot for containerization, which does not require root access on your Android device.

Can I install GUI applications?

Yes! Once inside the container, you can install any X11 or Wayland application via pacman:

pacman -S firefox gimp libreoffice vlc
How much storage does this use?

The base installation uses approximately 2-3GB. With additional applications, budget 5-10GB depending on what you install.

Is this the same as termux-x11 alone?

No. Termux-X11 provides basic X11 forwarding. LabWC Android adds a full Wayland compositor with GPU acceleration, providing a true desktop experience.

Can I use this for development?

Absolutely. The Arch Linux environment includes gcc, make, cmake, and other development tools. You can install VS Code, Git, and all your dev tools.
